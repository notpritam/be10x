Walkthrough — you create a task, and how your agent works it
  1. You create the task.website or agent Fill the form on the board (or your agent calls gfa_create_task). It lands in Backlog. Nothing runs yet — it's just captured & logged.
  2. You shape & approve it. Research → plan (you or the agent), tag a reviewer, approve → the task reaches Ready to work. That gate is deliberate: no work starts without an approved plan.
  3. Your agent picks it up — no copy‑paste. Because you ran be10x link, your agent in that repo already sees this exact task (same DB). It calls gfa_claim_task (or the be10x work runner auto‑claims) → status flips to In progress.
  4. It works in your repo & streams progress. As it goes it calls gfa_update_progress — the board's agent panel + activity feed update live. You never refresh or move anything by hand.
  5. It pauses to ask you. Hit a decision? gfa_request_input parks it in Needs input; you answer on the website; it resumes right where it left off.
  6. You supervise from the website. Approve reviews, answer inputs, watch it verify & finish — same data on both sides, always live.
There is no "sync" step. The website and the CLI/agent are two views of one SQLite DB. Create anywhere, work anywhere, watch anywhere — it's the same task, live on both.
