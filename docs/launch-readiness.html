V1 limitations — by design, documented, not blockers

Single-host
Board + agents run on one server. Perfect for one team hosting one board.
No distributed runners
Teammates running agents on their own machines over HTTPS is V2.
Host-level Claude auth
Dashboard-managed per-user Claude login is V2; V1 uses one host key.
One local-only footnote: the running demo needs a single serve restart to activate the durability layer — but a fresh Docker deploy already includes it (all committed), so it is not a launch blocker. Verdict: build is done; launch is an afternoon of ops.
