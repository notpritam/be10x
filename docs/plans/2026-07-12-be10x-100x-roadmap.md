## Roadmap (ordered: safe green wins first, then big-signal, then heavy, then polish)

1. **[DONE]** Environment / device metadata capture + card — browser, OS, screen, DPR,
   timezone, locale, connection, and page-load performance (TTFB/FCP/DCL/load) →
   `meta.environment`; dashboard + public "Environment" card.
2. **[DONE]** Auto error capture → auto-marker — uncaught errors + unhandled rejections become
   error-level console entries (already) AND drop a marker at the first error in a take, so the
   replay auto-highlights the failure moment. Dashboard already renders markers. Add a distinct
   auto-marker style (label prefix) + a bug-detail "N errors captured" chip.
3. **[DONE]** **Bug-detail MCP server (#7)** — `mcp-bug` stdio server exposing a filed bug's full
   capture as agent tools (list/get bug, console, network, dom, picked elements + notes,
   drawings, credentials, replay events, dom_at(t) via server-side rrweb rebuild, markers/visits,
   resources). Paste a bug id/URL → an agent debugs it autonomously. Model on the existing MCP.
4. **[DONE]** **Bug → Task handoff** — "Send to an agent to fix" on a bug → creates a
   git-for-agents task (title, description, bug link, picked component/source as scope) so the
   coding agent fixes it. Closes QA→fix loop. Reverse-link the task on the bug.
5. **[DONE]** **Capture everything (#6)** — beyond DOM/network/session: the page's rendered HTML
   source + all inline `<script>`/`<style>` text + a resource manifest (URL/type/size/timing from
   PerformanceResourceTiming). Bounded caps; a new `source.json` artifact + dashboard "Source &
   resources" panel. (No raw image bytes yet — rrweb already inlines much; manifest first.)
6. **[DONE]** **AI analysis / auto-RCA card** — server derives a heuristic root-cause summary from
   the capture (console errors + failed requests + picked component/source + error markers):
   suspected cause, evidence, repro steps, suspected file/component. Graceful, no external key
   required (deterministic heuristics); an LLM upgrade can layer on later.
7. **[DONE]** **Dashboard polish** — bug list free-text search + keyboard nav (↑/↓/↵), sort, and a
   result count; quick status actions; denser, richer detail layout.
8. **[DONE]** **Reporter polish** — widget keyboard shortcuts, better empty/enabled states,
   copyable selectors, and a "capture health" hint (what got captured) before filing.
